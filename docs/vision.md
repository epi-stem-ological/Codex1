# Vision

Codex1 exists to make AI-assisted software engineering more disciplined, reviewable, and reusable.

The central idea is simple:

> AI coding agents perform better when they are given clear task contracts, tight scope, explicit acceptance criteria, privacy boundaries, and reviewable handoffs.

This repo is not a private command center. It is the public-facing toolkit that can be reused by anyone who wants cleaner agentic coding workflows.

## What This Repo Should Become

Codex1 should become a practical public starter kit with:

- prompt contracts
- task specs
- issue templates
- handoff templates
- review templates
- agent workflow protocols
- small examples using fake data
- eventually, simple tools that generate and validate these artifacts

## What This Repo Should Not Become

Codex1 should not become:

- a dump of private prompts
- a personal journal
- a clone of private repos
- a place for employer-specific work
- a pile of untested scripts
- an overengineered framework before the workflow is proven

## Design Philosophy

1. **Structure beats memory.** Good files outlast good chats.
2. **Small tasks beat vague ambition.** Agents need crisp boundaries.
3. **Reviewability beats speed.** Fast garbage is still garbage.
4. **Public-safe by default.** If it cannot be safely public, it does not belong here.
5. **Templates before tooling.** Prove the workflow manually before automating it.
