# Public-Private Boundary

Codex1 is public. That means every artifact must be safe to share widely.

## Belongs Here

- generic agentic coding workflows
- public-safe templates
- fake examples
- reusable issue templates
- public software engineering protocols
- sanitized lessons learned
- small tools that do not require private data

## Does Not Belong Here

- private personal operating system content
- financial records
- tax records
- family details
- private contact lists
- employer-specific material
- client or customer data
- internal incident details
- private screenshots
- production credentials
- API keys or secrets
- real crypto wallet secrets or seed phrases

## Sanitization Standard

Before moving anything from a private repo into Codex1, verify that it has:

1. no real names unless intentionally public
2. no private numbers
3. no private addresses
4. no employer-confidential details
5. no private documents
6. no credentials or tokens
7. no real customer, vendor, or family data
8. no hidden metadata that exposes sensitive information

## Safe Example Pattern

Use examples like:

```text
Sample User
Sample Company
Sample Project
example.com
$1,000 sample amount
2026-01-01 sample date
```

## Rule of Thumb

If publishing it would make you hesitate, it does not belong in this repo.
